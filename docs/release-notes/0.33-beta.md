# Zarya 0.33.0-beta

## Focus

Stable release hardening.

## Added

- Stable scope definition (`docs/stable/`)
- Feature gating by release channel (`FeatureGate`)
- Stable release criteria, risk register, 1.0 backlog split
- Stable and experimental regression matrices
- Go/no-go checklist
- Diagnostics fields for release channel and experimental feature state
- `configuredRuntimeMode` vs `effectiveRuntimeMode` in diagnostics

## Changed

- Experimental TUN, helper, and kill switch are gated more strictly on stable channel
- Stable channel defaults to Xray system-proxy effective runtime
- Settings → Release channel controls experimental visibility

## Still experimental

- sing-box TUN
- zarya-helper
- Linux nftables kill switch
- Windows WFP kill switch

## 1.0 note

Production TUN is **not** required for 1.0. See [docs/stable/stable-scope.md](../stable/stable-scope.md).
