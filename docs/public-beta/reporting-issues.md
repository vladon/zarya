# Reporting Issues

## Before opening an issue

1. Update to the latest beta build.
2. Reproduce the problem.
3. Create a **Diagnostics Bundle**: **Help → Create Diagnostics Bundle**
4. Review the bundle before sharing.

## Do not share

- raw `vless://`, `vmess://`, `trojan://`, `ss://` links
- subscription URLs
- passwords
- private keys
- helper tokens
- unredacted generated configs

## Include

- platform and OS version
- Zarya version
- runtime mode
- core versions
- steps to reproduce
- expected result
- actual result
- diagnostics bundle

## GitHub

If the project issues URL is configured in your build, **Help → Report Issue** opens the issue chooser.
Otherwise use the bundled issue templates from the repository `.github/ISSUE_TEMPLATE/` directory.
