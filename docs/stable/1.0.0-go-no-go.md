## CI / release assets (2026-06-30)

| Platform | CI packaging | Status on v1.0.0 release |
| --- | --- | --- |
| Windows x64 portable | `release.yml` / `build.yml` + `package-windows.ps1` | Published |
| macOS arm64/x64 | `scripts/package-macos.sh` only (not in workflows) | Missing |
| Linux x64 | `scripts/package-linux.sh` only (not in workflows) | Missing |

**Workflow gaps:** Neither `build.yml` nor `release.yml` defines `workflow_dispatch`. `release.yml` is Windows-only and currently fails workflow validation (invalid `secrets` in `if:`). Tag `v1.0.0` did not produce downloadable macOS/Linux CI artifacts.

**Build failures on main** ([run 28411488533](https://github.com/vladon/zarya/actions/runs/28411488533)):

- **ubuntu:** compile error in `AppUpdateChecker.cpp` (`QUrl::setQuery({})` ambiguous with Qt 6 on Linux).
- **macos (arm64):** link failure — CI builds only `zarya` target; undefined symbols from `FeatureGate`, `FeaturePolicy`, `DefaultSettings`, `InstallationInfo` (helper/updater/feature objects not linked in CI job).

**Manual path until CI fixed:** On macOS/Linux hosts at tag `v1.0.0`, run full build + `package-macos.sh` / `package-linux.sh --skip-signing`, verify, `gh release upload v1.0.0`, refresh `SHA256SUMS.txt`.
